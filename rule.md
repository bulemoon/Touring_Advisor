# 开发规范

> 适用于人工开发和 AI 辅助开发的统一规范。

## 代码规范

- **简洁优先**：避免过度封装和冗余注释
- **Tool 原子化**：每个 Tool 只做一件事
- **接口契约**：所有 Tool 统一返回格式 `{"success": bool, "data": any, "error": str}`
- **缓存优先**：调用外部 API 前先查 Redis 缓存
- **环境变量**：API Key 一律放 .env，不硬编码，不提交到 Git

## AI 协作规则

- 需求不清楚时 → 停下来讨论
- 路径不优时 → 建议更好方法
- 追根因，不打补丁
- 修改 planner.py 或 Tool 接口前先说明影响范围

## 禁止事项
- 不在 Tool 内直接调用 LLM
- 不跳过缓存直接调知乎 API
- 不硬编码后端地址
- 不提交 .env 到 Git
