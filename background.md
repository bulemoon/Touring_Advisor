# 项目背景

## 项目名称
Touring Advisor

## 创建日期
2026-05-16

## 问题陈述
用户输入旅行计划（时间、地点、景点），系统通过 Agent 聚合多源信息，生成个性化旅行建议，包括景点攻略、天气适配装备推荐、购买链接和可交互地图，一站式解决旅行规划问题。

## 目标用户
- 计划出行的旅行者（国内用户为主）
- 使用 PC 或手机浏览器访问

## 核心功能
1. 行程输入：支持自由文本输入（时间、地点、景点，长短不限）
2. 多源内容聚合：通过知乎搜索/直答/全网搜索 API 获取景点攻略和旅行建议
3. 智能装备推荐：结合目的地天气、季节、地形推荐具体装备（充电宝、衣物、防晒、防虫等）
4. 电商购买链接：装备推荐旁标注购买入口（MVP 静态链接，后期接淘宝联盟）
5. 可交互地图：OSM 地图标注景点、周边店铺，支持点击查看详情
6. 响应式设计：兼容 PC 和手机屏幕

## 已知约束
- 部署需中国大陆可访问（选用 GitHub Pages + Zeabur 香港节点）
- MVP 阶段匿名使用，无用户登录
- 淘宝联盟账号待申请，MVP 期间使用静态商品链接
- 知乎 API 限流：全局 10 QPS，需做缓存

## 成功标准
- 用户输入行程后，30 秒内返回完整推荐结果
- 装备推荐与目的地天气/季节匹配准确
- 地图正确标注主要景点和周边信息
- 页面在主流手机浏览器下正常展示

## 项目范围
### 包含
- 单次行程规划（无历史记录）
- 国内旅行目的地
- 图文推荐 + 购买链接 + OSM 地图

### 不包含（Out of Scope）
- 用户账号和行程保存
- 国际旅行
- 机票/酒店预订
- 社交分享功能
